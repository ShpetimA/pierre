## Overview

<Notice variant="warning" icon={<IconCiWarningFill />}>
  File tree is in early active development—APIs are subject to change.
</Notice>

**File tree** is a library for rendering file trees on the web. It includes
vanilla JavaScript and React components, with support for SSR. You get flexible
components for building trees with filtering, directory flattening, icons, and
customizable styling.

<FileTree
  options={overviewFileTreeOptions}

  }}
/>

We have an opinionated stance in our architecture: **file trees benefit from
headless state management**. Our core path ingestion and tree store is highly
optimized for fast ingestion of large trees and near constant time mutations.
Most features are configurable, but are strongly integrated so that they can
exist together coherently and performantly.

`FileTree` is a vanilla javascript class that can inject trees into a container
on your page. We also expose a `@pierre/trees/react` implementation, which is
just a thin wrapper around the vanilla javascript class.

## Rendering Trees

The `FileTree` component is available in both vanilla JavaScript and React. You
pass a list of file paths (and optional [options](#core-types-filetreeoptions));
the tree builds the hierarchy and handles expand/collapse, selection, and
search. Examples for both environments are below.
