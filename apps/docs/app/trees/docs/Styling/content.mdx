## Styling

The file tree is rendered inside a shadow DOM host (the `file-tree-container`
custom element), so its styles are isolated from your page.

Customize it by passing `className` and `style` on the host: use those props on
the React `FileTree` component, or style the host returned by
`getFileTreeContainer()` after `render()` in vanilla JS. That lets you control
layout (`width`, `maxHeight`, borders, overflow) and override the tree’s CSS
variables from global CSS, a wrapper, or the host element directly.

<Notice icon={<IconBulbFill />}>
  Think of styling in two layers: host styles change the outer panel, and
  `--trees-*` variables change the UI rendered inside the shadow DOM.
</Notice>

Use `--trees-density-override` when you want one-knob spacing changes across
padding, gaps, indentation, border radius, and related layout tokens. Density
does not change row height on its own. Set `--trees-row-height-override`
separately when you want shorter or taller rows.

If you customize row height, keep `itemHeight` aligned with the same pixel
value. Virtualization still computes layout from the numeric `itemHeight`
option, so `itemHeight` and `--trees-row-height-override` should describe the
same row height whenever you move away from the default `30px`.

A good preset pattern is to pair the two knobs together:

- compact: `itemHeight: 24`, `--trees-row-height-override: 24px`,
  `--trees-density-override: 0.8`
- default: `itemHeight: 30`, `--trees-row-height-override: 30px`,
  `--trees-density-override: 1`
- relaxed: `itemHeight: 36`, `--trees-row-height-override: 36px`,
  `--trees-density-override: 1.2`

### Unsafe CSS escape hatch

If CSS variables are not enough, pass `unsafeCSS` in `FileTreeOptions` to inject
raw CSS into the tree shadow root. This is the same escape hatch pattern we use
in diffs, but for trees it should stay the exception rather than the default
approach.

### Fallback order

Trees uses a consistent fallback chain for colors and theme tokens:

1. `--trees-*-override`
2. `--trees-theme-*`
3. library defaults

That means explicit overrides always win. Theme-derived values from
`themeToTreeStyles()` sit in the middle, and the built-in defaults are only used
when neither of those is present.
