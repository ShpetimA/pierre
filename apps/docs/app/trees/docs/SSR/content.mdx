## SSR

<Notice icon={<IconBulbFill />}>
  Import SSR utilities from `@pierre/trees/ssr`.
</Notice>

The SSR API lets you pre-render the file tree HTML on the server for fast first
paint, then hydrate on the client for full interactivity.

### Usage

`preloadFileTree` returns a payload object:
`{ id, outerStart, domOuterStart, shadowHtml, outerEnd }`. Pass
`{ id, shadowHtml }` to the React `FileTree` as `preloadedData`. Use
`${payload.outerStart}${payload.shadowHtml}${payload.outerEnd}` when the HTML
parser sees the server markup directly, or use
`${payload.domOuterStart}${payload.shadowHtml}${payload.outerEnd}` when you need
to insert the full container through DOM APIs such as `dangerouslySetInnerHTML`
or `innerHTML`. For vanilla hydration, render the pre-rendered container into
the page and call `hydrate({ fileTreeContainer })`.

<Notice variant="warning" icon={<IconCiWarningFill />}>
  Options used for pre-rendering must exactly match what you pass on the client.
  Use the same `FileTreeOptions` object (or equivalent) when calling
  `preloadFileTree` and when rendering `FileTree` so the tree structure and
  state hydrate correctly.
</Notice>

### Preloaders

#### preloadFileTree [toc-ignore]

Preloads the file tree HTML from a `FileTreeOptions` object and an optional
`FileTreeStateConfig` (for initial expanded/selected items). Use this in a
server component or server route; pass the returned payload and the same options
to your client component. Pass `{ id, shadowHtml }` to the React `FileTree` as
`preloadedData`, or concatenate
`${payload.domOuterStart}${payload.shadowHtml}${payload.outerEnd}` before
calling vanilla `hydrate()` with the server-rendered container.
