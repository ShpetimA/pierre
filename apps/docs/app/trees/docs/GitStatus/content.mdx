## Git Status

Use the `gitStatus` option to show file-level status indicators for added,
modified, deleted, renamed, untracked, and ignored items. Folders that contain
changed descendants automatically receive a dot hint, and ignored rows use
ignored styling instead of a text badge. [Try the live demo](../#git-status) on
the homepage.

`GitStatusEntry` pairs a file `path` with a status value:
`{ path: string; status: 'added' | 'modified' | 'deleted' | 'renamed' | 'untracked' | 'ignored' }`.

Import `GitStatusEntry` from `@pierre/trees` and pass an array via `gitStatus`
in constructor options (vanilla) or as a top-level prop (React). `path` should
match your tree file paths (for example, `src/index.ts`). Directory paths can
use a trailing slash (for example, `build/`) when you want folder-level ignored
styling to inherit onto descendants.

In vanilla JS you can also update status imperatively:

- `fileTree.setGitStatus(entries | undefined)` updates statuses and re-renders.
- `fileTree.getGitStatus()` returns the current `GitStatusEntry[] | undefined`.

When `gitStatus` is provided, tree rows include status-aware attributes you can
target:

- `data-item-git-status="added" | "modified" | "deleted" | "renamed" | "untracked" | "ignored"`
  on rows with direct or inherited Git status styling
- `data-item-contains-git-change="true"` on folders that contain changed
  descendants and render the dot indicator
